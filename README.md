## Технічні особливості реалізації

* **Переклад**: Реалізовано повну підтримку мультимовності. Зміна мови в селекторі автоматично оновлює контент без
  перезавантаження сторінки.

* **Зміна кольорів**:
    * Впроваджено систему динамічної зміни стилів залежно від обраного регіону.
    * На даному етапі реалізовано централізований конфіг кольорів, де ключові акценти змінюються автоматично (наприклад,
      для UA-локації застосовуються специфічні брендові кольори).
    * Архітектура дозволяє легко масштабувати список кольорів та додавати нові схеми для будь-якого Geo в один файл
      конфігурації.

* **Background**: Оскільки в ТЗ не було вказано специфічних вимог до реалізації фону (анімація, тощо), було використано
  статичне зображення. Це забезпечує стабільну швидкість завантаження сторінки та повну відповідність
  наданому макету.

* **Iframe (Open Game)**:
    * Реалізовано запуск гри через модальне вікно і в ньому **Iframe**.
    * Це рішення дозволяє утримувати користувача на платформі, забезпечуючи безшовний перехід
      до ігрового процесу без відкриття нових вкладок або сторінок.

* Код побудований на принципах **DRY** та **Single Responsibility**.
* Використано **Feature-Sliced Design (FSD)** для чіткої структуризації логіки, UI та конфігурацій.
